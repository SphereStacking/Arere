# Getting Started

`arere` is a TypeScript-based persistent terminal launcher. This guide covers the basics of using `arere`.

## Installation

### Install with npm (Recommended)

```bash
npm install -g arere
```

### Other Package Managers

```bash
# yarn
yarn global add arere

# pnpm
pnpm add -g arere

# npx (no installation required, downloads each time)
npx arere
```

### Requirements

- Node.js >= 18.0.0
- npm >= 9.0.0
- OS: macOS, Linux, Windows (WSL recommended)

## Create Your First Action

Actions are placed in the following directories (create them if needed):

- **Global actions**: `~/.arere/actions/`
- **Project-specific actions**: `./.arere/actions/`

### Method 1: Auto-generate from Template (Recommended)

Using the `arere-plugin-create` plugin, you can create actions interactively:

```bash
# Install the plugin
npm install -g arere-plugin-create

# Launch arere
arere
```

Select `create-action` to interactively configure:

1. **Action name** - Becomes the filename (e.g., `my-action`)
2. **Description** - Description shown in the action list
3. **Category** - Choose from utility, development, git, testing, demo
4. **Tags** - Select multiple tags for searching
5. **Template** - Choose from basic, advanced, with-i18n
6. **Save location** - workspace (project) or global

Once configured, the action file is automatically generated.

### Method 2: Create Manually

You can also create files manually without using templates:

```bash
mkdir -p ~/.arere/actions
```

```typescript
// ~/.arere/actions/hello.ts
import { defineAction } from 'arere'

export default defineAction({
  name: 'hello',
  description: 'Display Hello World',
  async run({ tui }) {
    const name = await tui.prompt.text('What is your name?')
    tui.output.success(`Hello, ${name}!`)
  }
})
```
