# What are Plugins

Plugins are a mechanism for distributing and sharing actions as npm packages.

## Benefits of Plugins

- **Reusability**: Use the same actions across multiple projects
- **Sharing**: Share with other developers via npm
- **Dependency Management**: Plugins can have their own dependencies
- **Version Control**: Manage versions with semantic versioning

## Plugin Naming Convention

Plugin names must start with `arere-plugin-*`:

- `arere-plugin-git`
- `arere-plugin-docker`
- `arere-plugin-aws`
- ❌ `git-plugin` (prefix is required)
- ❌ `arere-git` (must explicitly indicate it's a plugin)

## Plugin Structure

Basic plugin structure:

```
arere-plugin-example/
├── package.json
├── tsconfig.json
├── tsup.config.ts
├── src/
│   └── index.ts          # Plugin definition
├── actions/
│   ├── hello.ts          # Action 1
│   └── goodbye.ts        # Action 2
├── locales/              # Translation files (optional)
│   ├── en/translation.json
│   └── ja/translation.json
└── README.md
```
