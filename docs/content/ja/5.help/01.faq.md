# FAQ

よくある質問と回答をまとめています。

## 一般的な質問

### arere とは何ですか？

arere は TypeScript ベースの常駐型ターミナルランチャーです。よく使うコマンドやスクリプトを「アクション」として定義し、対話的に実行できます。

### どのような用途に向いていますか？

- 開発ワークフローの効率化（Git操作、デプロイなど）
- 繰り返し行う作業の自動化
- チーム内でのスクリプト共有（プラグインとして配布）
- CI/CD パイプラインでの利用

### 必要なシステム要件は？

- Node.js 18.0.0 以上
- npm 9.0.0 以上
- macOS、Linux、Windows (WSL推奨)

## インストールについて

### グローバルインストールとローカルインストールの違いは？

**グローバルインストール**（推奨）：
```bash
npm install -g arere
```
どのディレクトリからでも `arere` コマンドを実行できます。

**ローカルインストール**：
```bash
npm install arere
```
プロジェクト内でのみ使用でき、`npx arere` で実行します。

### npx で毎回実行しても良いですか？

可能ですが、毎回パッケージのダウンロードが発生するため遅くなります。頻繁に使う場合はグローバルインストールを推奨します。

## アクションについて

### アクションはどこに配置すればいいですか？

- **グローバルアクション**: `~/.arere/` - すべてのプロジェクトで使用可能
- **プロジェクト固有**: `./.arere/` - そのプロジェクトでのみ使用可能

### TypeScript 以外で書けますか？

現在は TypeScript のみサポートしています。アクションは実行時に自動的にトランスパイルされます。

### アクションの name は必須ですか？

必須ではありません。省略した場合、ファイル名から自動的に推論されます。

```typescript
// ~/.arere/hello-world.ts
export default defineAction({
  // name: 'hello-world' が自動設定される
  description: 'Hello World',
  async run() { ... }
})
```

### アクションのテストはどう書けばいいですか？

通常の TypeScript/JavaScript のテストフレームワーク（Jest、Vitest など）が使えます。

```typescript
import { describe, it, expect } from 'vitest'
import action from '../actions/hello'

describe('hello action', () => {
  it('should have correct name', () => {
    expect(action.name).toBe('hello')
  })
})
```

## プラグインについて

### プラグインの命名規則は？

プラグイン名は必ず `arere-plugin-` で始める必要があります。

- ✅ `arere-plugin-git`
- ✅ `arere-plugin-docker`
- ❌ `git-plugin`

### プラグインを公開せずに使えますか？

はい。`npm link` を使ってローカルでリンクできます：

```bash
cd arere-plugin-my-plugin
npm run build
npm link

# 使用するプロジェクトで
npm link arere-plugin-my-plugin
```

### プラグインの依存関係はどう管理しますか？

プラグイン固有の依存関係は `dependencies` に、arere 本体は `peerDependencies` に指定します：

```json
{
  "dependencies": {
    "date-fns": "^2.30.0"
  },
  "peerDependencies": {
    "arere": ">=0.1.0"
  }
}
```

## CI/CD について

### CI/CD 環境で使えますか？

はい。`arere run <action>` でヘッドレスモード（UIなし）で実行できます。

```yaml
# GitHub Actions
- run: npx arere run deploy
```

### 環境変数はどう渡しますか？

アクション内で `env` オブジェクトからアクセスできます：

```typescript
export default defineAction({
  async run({ env }) {
    const apiKey = env.API_KEY
    // ...
  }
})
```

## 国際化について

### どの言語がサポートされていますか？

現在は英語（en）と日本語（ja）がサポートされています。

### 言語の切り替え方法は？

環境変数 `LANG` または設定ファイルで言語を指定できます。

### プラグインの翻訳を追加するには？

プラグインディレクトリに `locales/` フォルダを作成し、言語ごとの翻訳ファイルを配置します：

```
locales/
├── en/translation.json
└── ja/translation.json
```

詳細は [設定スキーマ](/ja/1.guides/04.plugins/03.config) を参照してください。

## その他

### ログはどこに保存されますか？

デフォルトではログは保存されません。デバッグが必要な場合は、`DEBUG=arere*` 環境変数を設定してください。

### キャッシュをクリアするには？

```bash
rm -rf ~/.cache/arere
```

### バグを報告するには？

GitHub Issues で報告してください：
