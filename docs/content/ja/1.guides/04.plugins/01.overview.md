# プラグインとは

プラグインは、アクションをnpmパッケージとして配布・共有する仕組みです。

## プラグインのメリット

- **再利用性**: 複数のプロジェクトで同じアクションを使える
- **共有**: npmで他の開発者と共有できる
- **依存管理**: プラグイン独自の依存関係を持てる
- **バージョン管理**: セマンティックバージョニングでバージョン管理

## プラグインの命名規則

プラグイン名は `arere-plugin-*` で始める必要があります：

- `arere-plugin-git`
- `arere-plugin-docker`
- `arere-plugin-aws`
- ❌ `git-plugin` (プレフィックスが必須)
- ❌ `arere-git` (プラグインであることを明示)

## プラグインの構造

基本的なプラグインの構造：

```
arere-plugin-example/
├── package.json
├── tsconfig.json
├── tsup.config.ts
├── src/
│   └── index.ts          # プラグイン定義
├── actions/
│   ├── hello.ts          # アクション1
│   └── goodbye.ts        # アクション2
├── locales/              # 翻訳ファイル（オプション）
│   ├── en/translation.json
│   └── ja/translation.json
└── README.md
```
